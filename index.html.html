The Scenario

The Problem: Patry's property management team receives approximately 50 tenant maintenance requests per week via email. Each request is currently logged manually into a spreadsheet, categorized by urgency and trade type, and assigned to a contractor via a follow-up email. The process takes a staff member roughly 6–8 hours per week and is prone to delays and missed requests.

You have been given access to 2.5 weeks of real intake data from this process — 51 rows of tenant maintenance requests and a contractor directory of 10 tradespeople. The data is intentionally messy. It reflects what actually arrives in a property management inbox.

🎯

Your Task

Build a working n8n workflow that automates the maintenance request process end-to-end. Your workflow must complete all five steps below:

1
Read the Maintenance Inbox
Connect to the provided Google Sheet and read incoming maintenance request rows.
2
Classify each request using AI
Use the provided API via an HTTP Request node to extract the urgency level (Emergency or Standard) and trade type (Plumbing, Electrical, HVAC, General, Appliance, Roofing/Structure) from the free-text description. Your prompt must ask the model to explain its urgency reasoning — not just output a label. Configuration instructions and a reference snippet are provided in the setup section below.
3
Match to the correct contractor
Look up the Contractor Directory and assign the right contractor based on both trade type AND urgency tier. Emergency requests must go to an Emergency-tier contractor — not a Standard-tier one.
4
Send or simulate a contractor notification
Either send a live email or Slack message to the assigned contractor using your own credentials, OR use an n8n Set node to construct the notification payload. Both are acceptable.
5
Log every outcome to your Output Log
Every row — completed, skipped (duplicate), or errored (missing data) — must produce a row in your personal Output Log sheet.

Anthropic API — HTTP Request Node Configuration

This assessment uses the Anthropic API via n8n's HTTP Request node rather than a pre-built AI node. This is intentional — configuring a raw API call is part of what is being assessed. The reference below gives you the exact structure. How you integrate it into your workflow, prompt it, and parse its output is up to you.

Model to use: claude-haiku-4-5-20251001
Your API key: Provided in your welcome email. Copy it exactly — do not add spaces or line breaks.

HTTP Request Node Settings:

Method: POST URL: https://api.anthropic.com/v1/messages Headers: x-api-key: [YOUR API KEY FROM WELCOME EMAIL] anthropic-version: 2023-06-01 content-type: application/json Body (JSON): { "model": "claude-haiku-4-5-20251001", "max_tokens": 300, "messages": [ { "role": "user", "content": "You are a property management assistant. Classify the following maintenance request.\n\nRequest: {{ $json.description }}\n\nReturn a JSON object with exactly these fields:\n- urgency: either 'Emergency' or 'Standard'\n- trade_type: one of 'Plumbing', 'Electrical', 'HVAC', 'General', 'Appliance', 'Roofing/Structure'\n- reasoning: one sentence explaining your urgency classification\n\nReturn only valid JSON. No preamble or explanation outside the JSON object." } ] }

Parsing the Response:

The API returns a JSON object. Your classification result is a text string nested inside it at:

{{ $json.content[0].text }}

That string is itself a JSON object — so you need to convert it into separate values before the rest of your workflow can use urgency, trade_type, and reasoning. Add a Code node immediately after the HTTP Request node and paste in the following:

const raw = $input.item.json.content[0].text; const result = JSON.parse(raw); return [{ json: result }];

After this node runs, urgency, trade_type, and reasoning will be available as individual fields you can reference in downstream nodes like any other value. If you prefer not to use a Code node, a Set node with Expression mode can also extract individual fields using expressions like {{ JSON.parse($json.content[0].text).urgency }}.

💡 Testing tip: Run the HTTP Request node and Code node together on a single test row first. Open the output panel and confirm you can see urgency, trade_type, and reasoning as separate fields before building the rest of your workflow.
